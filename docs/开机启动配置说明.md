# 开机启动配置说明

## `start.bat` 是做什么的？

`start.bat` 是项目的**交互式一键启动脚本**，主要完成以下工作：

1. 检查 Python / pip 环境
2. 自动创建并激活 `venv` 虚拟环境
3. 安装 `requirements.txt` 中的依赖
4. 检查或生成 `.env` 配置文件
5. 输出访问地址（管理后台、登录页、API 文档等）
6. 最后执行 `python app.py` 启动 FastAPI 服务

它适合**首次部署或手动启动**，但因为它会打开 `cmd /k` 窗口并需要交互，所以**不适合直接放到开机启动里**。

---

## 开机自启文件说明

项目中已生成以下用于开机自启的文件：

| 文件 | 作用 |
|------|------|
| `start_hidden.bat` | 后台运行服务，无弹窗，输出写入 `logs\startup.log` |
| `startup.vbs` | 隐藏窗口调用 `start_hidden.bat`（启动文件夹方案用） |
| `setup_startup_task.bat` | 计划任务方案入口（内部调用 `setup_startup_task.ps1`） |
| `setup_startup_task.ps1` | 计划任务安装脚本（自动 UAC 提权 + 注册 SYSTEM 任务 + 全程日志 `logs\setup_task.log`） |
| `wechat-autostart.xml` | 计划任务 XML 定义模板（含路径占位符，项目搬迁不失效） |

---

## 方式一：启动文件夹（最简单）

### 配置开机启动

1. 按 `Win + R`，输入 `shell:startup`，回车。
2. 将项目根目录下的 `startup.vbs` 复制到打开的启动文件夹中。
3. 重启电脑，登录后服务会自动在后台启动。
4. 服务日志查看：`logs\startup.log`

### 取消开机启动

1. 按 `Win + R`，输入 `shell:startup`，回车。
2. 找到 `startup.vbs`，右键删除。

> 如果也复制到了公共启动文件夹，可按 `Win + R` 输入 `shell:common startup` 检查并删除。

---

## 方式二：Windows 计划任务（更稳定，推荐）

> 本方案以 `SYSTEM` 身份运行，**开机即启动（无需登录）**，并支持**崩溃自动重启**。
> ⚠️ 注意：方式一与方式二**不要同时启用**，否则可能同时启动两个实例导致端口冲突。

### 配置开机启动

1. 双击 `setup_startup_task.bat`（脚本会自动弹 UAC 提权窗口，点“是”）。
2. 提权后的 PowerShell 窗口会自动完成注册，显示 `[OK] Task ... created successfully!` 即成功。
3. 任务名固定为 `WeChatDownloadAPI_AutoStart`，特性如下：
   - `BootTrigger` 系统启动时触发 + `LogonTrigger` 登录时触发（双保险）
   - 崩溃后每 60 秒自动重启，最多 3 次
   - 以 `SYSTEM` 身份运行，隐藏窗口
4. 安装日志：`logs\setup_task.log`，服务日志：`logs\startup.log`

### 取消开机启动

删除 SYSTEM 计划任务需要管理员权限，任选其一：

**方法 A：命令行**

以**管理员身份**打开 CMD 或 PowerShell，执行：

```cmd
schtasks /Delete /TN WeChatDownloadAPI_AutoStart /F
```

看到 `成功: 计划任务 WeChatDownloadAPI_AutoStart 被成功删除` 即完成。

**方法 B：图形界面**

`Win + R` 输入 `taskschd.msc` 打开**任务计划程序** → 任务计划程序库 → 找到 `WeChatDownloadAPI_AutoStart` → 右键 → **删除**。

### 验证是否已删除

```cmd
schtasks /Query /TN WeChatDownloadAPI_AutoStart
```

若提示“错误: 找不到指定的任务”或“拒绝访问”（后者也可能因权限，配合图形界面确认），说明已删除。

## 停止当前正在运行的服务

取消开机启动**不会**停止当前已运行的服务。若要立即停掉正在运行的服务：

- 打开 **任务管理器** → 找到 `python.exe` 进程 → 结束任务。
- 或在 CMD 中执行（精确按 PID 停止，不影响其他 Python 程序）：

```cmd
:: 先查到监听 5000 端口的 PID
netstat -ano | findstr ":5000" | findstr "LISTENING"

:: 假设上一步查到 PID 为 12345
taskkill /F /PID 12345
```

> 注意：`taskkill /F /IM python.exe` 会结束所有 Python 进程。如果电脑上还有其他 Python 程序在运行，建议用上面的按 PID 方式精确结束。

---

## 注意事项

- **首次部署请先用 `start.bat`**，完成环境配置、依赖安装和微信扫码登录。
- **方式一与方式二二选一**，不要同时启用，避免双开端口冲突。
- 开机自启前请确认 `.env` 中已有有效微信登录凭证，否则服务启动后仍需手动登录。
- 服务默认监听 `http://localhost:5000`。
- 如果项目搬迁了目录，重新运行一次 `setup_startup_task.bat` 即可（脚本会用 `%~dp0` 自动定位新路径，旧任务会被覆盖删除）。
