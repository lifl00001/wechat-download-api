# 投资日报数据采集方案

> 目标：按分类（AI、机器人、半导体等）采集网络新闻 → 存入数据库 → 喂给大模型生成投资日报

---

## 一、RSS 基础概念

### 1.1 RSS 是什么

RSS 是一种**标准化的文章列表格式**（XML）。一个 URL 返回最新的文章列表，包含标题、链接、摘要、时间，全都是结构化的。

```xml
<rss>
  <channel>
    <title>财联社电报</title>
    <item>
      <title>OpenAI发布GPT-5 多模态能力大幅提升</title>
      <link>https://www.cls.cn/detail/1234567</link>
      <description>OpenAI今日正式发布GPT-5模型...</description>
      <pubDate>Wed, 04 Jun 2026 09:30:00 +0800</pubDate>
    </item>
    <item>
      <title>特斯拉人形机器人Optimus开始小批量交付</title>
      <link>https://www.cls.cn/detail/1234568</link>
      <description>特斯拉宣布其人形机器人Optimus...</description>
      <pubDate>Wed, 04 Jun 2026 09:15:00 +0800</pubDate>
    </item>
  </channel>
</rss>
```

**本质：一个 URL 返回最新的文章列表。** 每隔一段时间请求这个 URL，就能拿到最新发布的文章。

### 1.2 现实问题

不是所有网站都提供 RSS：

| 网站 | 有原生 RSS 吗 | 原生 RSS 地址 |
|------|:---:|---|
| 虎嗅 | ✅ 有 | `https://www.huxiu.com/rss/0.xml` |
| 36氪 | ❌ 没有 | — |
| 财联社 | ❌ 没有 | — |
| 金十数据 | ❌ 没有 | — |
| 东方财富 | ❌ 没有 | — |

## 二、RSSHub 是什么

### 2.1 定位

RSSHub 是一个**"万能 RSS 转换器"**：给它一个网站名，它帮你实时爬取内容，然后以标准 RSS 格式输出。

```
没有 RSSHub：
  36氪 ──→ 自己写爬虫解析 HTML ──→ 提取文章列表
  财联社 ──→ 自己写爬虫解析 HTML ──→ 提取文章列表
  金十 ──→ 自己写爬虫解析 HTML ──→ 提取文章列表
  （每个网站一套代码，维护成本极高）

有了 RSSHub：
  36氪 ──→ RSSHub 帮你爬 ──→ 输出标准 RSS ──→ 你只需读 RSS
  财联社 ──→ RSSHub 帮你爬 ──→ 输出标准 RSS ──→ 你只需读 RSS
  金十 ──→ RSSHub 帮你爬 ──→ 输出标准 RSS ──→ 你只需读 RSS
  （你只需要一套代码：读 RSS 的代码）
```

### 2.2 数据存储

**RSSHub 不持久化存储新闻数据。** 每次请求都是实时去目标网站爬取，爬完即返回，不存历史。

```
请求流程：
  你 ──GET──→ RSSHub ──→ 实时去目标网站抓取 ──→ 解析成 RSS/XML ──→ 返回给你
```

内部只有短期缓存（默认 5 分钟，Redis 或内存），避免同一秒内多人请求同一源时重复爬取。

| 问题 | 答案 |
|------|------|
| RSSHub 有数据库存新闻吗？ | 没有 |
| 能查"昨天财联社发了什么"吗？ | 不能，只返回当前能爬到的 |
| 有 OpenAPI 可以按关键词搜索吗？ | 没有 |
| 能返回 JSON 吗？ | 能，加 `?format=json`，但本质还是"当前这一批" |

### 2.3 部署方式

Docker 自建（推荐）：

```yaml
# docker-compose.yml 中追加
rsshub:
  image: diygod/rsshub:latest
  ports:
    - "1200:1200"
  environment:
    NODE_ENV: production
    CACHE_TYPE: memory
    CACHE_EXPIRE: 300
  restart: always
```

启动后 `http://localhost:1200` 即为 RSSHub 实例。也可直接用公共实例 `https://rsshub.app`，但不保证稳定性。

## 三、采集数据的方式对比

### 3.1 八种方式总览

| 方式 | 返回格式 | 数据完整度 | 成本 | 维护成本 |
|------|----------|-----------|------|----------|
| ① **Tavily Search** | JSON（标题+URL+全文+时间+评分） | ⭐⭐⭐⭐⭐ 有全文 | 1000次/月免费 | 极低 |
| ② **Jina AI** | Markdown/JSON（标题+URL+正文） | ⭐⭐⭐⭐ 有全文 | 1000万token免费 | 极低 |
| ③ **SerpApi** | JSON（标题+URL+snippet） | ⭐⭐ 只有摘要 | 100次/月免费 | 低 |
| ④ RSSHub → XML/JSON | XML 或 JSON | 只有摘要 | 免费（自建） | 低（社区维护） |
| ⑤ RSSHub → 全文模式 | JSON | 有全文 | 免费（自建） | 低，但速度慢 |
| ⑥ 直接调各网站 API | JSON | 最完整（全文+标签+关联个股） | 免费 | 中（需自己对接） |
| ⑦ 搜索引擎 API (Bing等) | JSON | 只有摘要 | 免费额度有限 | 低 |
| ⑧ 浏览器自动化 (Playwright MCP等) | 自定义 | 看到啥提啥 | 高（LLM token费） | 高（不稳定） |

### 3.2 各方式详细说明

#### 方式 1：Tavily — 专为 AI Agent 设计的搜索引擎 ⭐推荐

Tavily 的定位是**"给大模型用的搜索引擎"**。搜索全网，返回干净、结构化的结果，直接可以喂给大模型。

四个核心 API：

| API | 做什么 | 消耗 |
|-----|--------|------|
| **Search** | 按关键词搜索，返回标题+URL+正文 | 1 credit (basic) / 2 credits (advanced) |
| **Extract** | 给 URL 列表，提取网页正文 | 每5个成功提取 = 1 credit |
| **Map** | 发现网站结构（站点地图） | 每10页 = 1 credit |
| **Research** | 多轮自动搜索+分析（类似 Perplexity） | 4~250 credits/次 |

**搜索新闻：**

```python
from tavily_python import TavilyClient

client = TavilyClient(api_key="tvly-xxxxx")

results = client.search(
    query="AI 投资 机器人 最新动态",
    topic="news",              # ← 关键：news 模式专搜新闻
    days=1,                    # 只看最近1天
    max_results=10,
    include_raw_content=True,  # 包含全文（不只是摘要）
)

for r in results["results"]:
    print(r["title"])           # "OpenAI发布GPT-5..."
    print(r["url"])             # "https://www.cls.cn/detail/..."
    print(r["content"])         # 干净的摘要（200-500字）
    print(r["raw_content"])     # ← 完整正文
    print(r["published_date"])  # "2026-06-04T09:30:00"
    print(r["score"])           # 相关度评分 0-1
```

**提取指定 URL 的正文：**

```python
extracted = client.extract(
    urls=[
        "https://www.cls.cn/detail/1800001",
        "https://36kr.com/p/1234567",
    ]
)
for e in extracted["results"]:
    print(e["url"])
    print(e["raw_content"])  # 干净的全文文本
```

**价格：**

| 方案 | 额度 | 月费 | 单价 |
|------|------|------|------|
| **免费 (Researcher)** | 1,000 credits/月 | $0 | — |
| Project | 4,000 credits/月 | $30 | $0.0075/credit |
| Bootstrap | 15,000 credits/月 | $100 | $0.0067/credit |
| 按量付费 | 无限 | — | $0.008/credit |

> 每天采集 3 个分类 × 1 次搜索 = ~90 credits/月，免费额度完全够用。

**优点：** 搜索+正文提取一步到位；有 news 专用模式；返回结果含相关度评分
**缺点：** 免费额度 1000 次/月，大规模使用需付费

#### 方式 2：Jina AI — 网页内容转 Markdown ⭐推荐

Jina 的核心能力：**给一个 URL，返回干净的 Markdown 正文**。还有搜索能力（`s.jina.ai`）。

**① `r.jina.ai` — 读网页（URL → Markdown）：**

```python
import requests

resp = requests.get(
    "https://r.jina.ai/https://www.cls.cn/detail/1800001",
    headers={
        "Accept": "application/json",
        "Authorization": "Bearer your-jina-api-key"
    }
)
data = resp.json()
print(data["data"]["title"])    # 文章标题
print(data["data"]["content"])  # 干净的 Markdown 正文
```

**② `s.jina.ai` — 搜索 + 自动提取前5条结果的正文：**

```python
resp = requests.get(
    "https://s.jina.ai/AI 投资 机器人 最新",
    headers={
        "Accept": "application/json",
        "Authorization": "Bearer your-jina-api-key"
    }
)
data = resp.json()
for item in data["data"]:
    print(item["title"])     # 标题
    print(item["url"])       # URL
    print(item["content"])   # 正文 Markdown
```

**③ DeepSearch — 深度搜索（多轮搜索+推理，类似 Perplexity）：**

```python
resp = requests.post(
    "https://deepsearch.jina.ai/v1/chat/completions",
    headers={"Authorization": "Bearer your-jina-api-key"},
    json={
        "model": "jina-deepsearch-v1",
        "messages": [{"role": "user", "content": "最近AI投资领域有什么重大事件？"}]
    }
)
# 自动多轮搜索+分析，返回深度答案
```

**价格：**

| 方案 | 限制 | 费用 |
|------|------|------|
| **无 API Key** | 20 RPM | $0 |
| **免费 API Key** | 500 RPM + **1000万免费 token** | $0 |
| **付费 API Key** | 500 RPM + 200万 TPM | 按 token 计费 |
| **Premium** | 5000 RPM | 更高价格 |

> 注册就送 1000 万 token，免费额度非常大。

**优点：** 免费额度大；Markdown 天然适合喂给大模型；`s.jina.ai` 搜索+提取一步完成
**缺点：** 搜索质量不如 Tavily；`s.jina.ai` 每次只返回5条

#### 方式 3：SerpApi — 搜索引擎结果接口

SerpApi 是**搜索引擎的代理**——帮你查 Google/Baidu/Bing，把搜索结果页解析成 JSON 返回。但**不提取正文**。

```python
from serpapi import GoogleSearch

search = GoogleSearch({
    "engine": "google_news",
    "q": "AI 投资 机器人",
    "gl": "cn",
    "hl": "zh-cn",
    "api_key": "your-serpapi-key"
})
result = search.get_dict()

for news in result["news_results"]:
    print(news["title"])    # ✅ 有
    print(news["link"])     # ✅ 有
    print(news["snippet"])  # ⚠️ 只有搜索引擎的2-3句摘要
    print(news["date"])     # ✅ 有
    # ❌ 没有正文全文！
```

**价格：**

| 方案 | 搜索次数 | 月费 |
|------|----------|------|
| **免费** | 100 次/月 | $0 |
| Production | 15,000 次/月 | $150/月 |
| Big Data | 30,000 次/月 | $275/月 |

**优点：** 支持最精确的搜索引擎控制（80+搜索引擎）；有 google_news 专用引擎
**缺点：** 最贵；免费额度最少；只有摘要没有全文，需二次调用 Jina/Tavily 提取

#### 方式 4：RSSHub → RSS/XML

```python
import feedparser
import httpx

resp = httpx.get("http://localhost:1200/cls/telegraph")
feed = feedparser.parse(resp.text)
for entry in feed.entries:
    print(entry.title, entry.link, entry.summary)
```

**优点：** 一个 URL 搞定一个源，不用自己写爬虫
**缺点：** 只有摘要，往往没有全文；需要自己存

#### 方式 2：RSSHub → JSON

```bash
curl "http://localhost:1200/cls/telegraph?format=json"
```

返回：

```json
{
  "title": "财联社｜电报",
  "items": [
    {
      "title": "OpenAI发布GPT-5",
      "url": "https://www.cls.cn/detail/1800001",
      "content_html": "<p>OpenAI今日正式发布...</p>",
      "summary": "OpenAI今日正式发布GPT-5模型...",
      "date_published": "2026-06-04T09:30:00+08:00",
      "author": "财联社"
    }
  ]
}
```

**优点：** JSON 直接就是 Python 字典，不用解析 XML
**缺点：** 和方式 1 本质一样，只是格式不同

#### 方式 3：RSSHub → 全文模式

```bash
curl "http://localhost:1200/cls/telegraph?mode=fulltext&format=json"
```

**优点：** `content_html` 会是完整文章内容，而不只是摘要
**缺点：** 慢很多（每条新闻都要额外请求一次原文页面）；有些网站全文抓取失败

#### 方式 4：直接调各网站 JSON API

```python
# 财联社电报 API（非官方，从浏览器开发者工具抓取）
resp = requests.get("https://www.cls.cn/nodeApi/telegraphList", params={
    "rn": 20,
    "last_time": "",
})
data = resp.json()

# 金十数据 API
resp = requests.get("https://flash-api.jin10.com/get_flash_list", params={
    "channel": "-8200",
    "vip": 1,
    "num": 20,
})
data = resp.json()
```

**优点：** 数据最完整（有全文、有分类标签、有时间戳）；响应快；结构化 JSON
**缺点：** 每个网站 API 不同，得自己找、自己对接；非官方 API 可能随时变

#### 方式 5：搜索引擎 API (Bing等)

```python
# Bing News Search API
resp = requests.get("https://api.bing.microsoft.com/v7.0/news/search",
    headers={"Ocp-Apim-Subscription-Key": "your-key"},
    params={"q": "AI 投资 机器人", "mkt": "zh-CN", "count": 50}
)
data = resp.json()
```

**优点：** 一个接口覆盖全网；按关键词搜索，天然支持"分类"需求
**缺点：** 免费额度有限（1000次/月）；只有摘要没有全文

#### 方式 8：大模型浏览器自动化

大模型通过浏览器自动化工具（Playwright MCP、Browser Use 等）控制浏览器，实现搜索、点击、提取。

| 工具 | 原理 | 特点 |
|------|------|------|
| **Playwright MCP** | LLM 通过 MCP 协议控制 Playwright 浏览器 | 微软官方，最成熟 |
| **Browser Use** | Python 库，大模型直接控制浏览器 | 开源，社区活跃 |
| **Claude Computer Use** | Claude 直接看屏幕截图操作 | Anthropic 官方 |
| **OpenAI Operator** | GPT 控制浏览器执行任务 | OpenAI 官方 |

```python
# Browser Use 示例
from browser_use import Agent
agent = Agent(task="搜索今天AI投资领域的重要新闻，提取标题和摘要")
result = await agent.run()
# 大模型会自己打开浏览器 → 搜索 → 点击 → 提取 → 返回结果
```

**优点：** 最灵活，什么网站都能处理；能处理需要登录的网站
**缺点：** 速度极慢（30秒~5分钟/任务）；成本高（~114K tokens ≈ $0.3-0.5/次）；不稳定（页面改版会失败）
**结论：** 适合临时查特定信息，**不适合定时批量采集**

## 九、与现有项目的关系

### 9.1 现有架构

```
微信公众号 (MP API) ──→ rss_poller ──→ rss_store (SQLite) ──→ RSS XML 输出
```

现有 `rss_poller` 硬编码调用微信 API，无法直接用于其他数据源。

### 9.2 扩展方案

新增一个**通用数据采集器**，与现有微信 poller 并行运行：

```
┌─── 现有（不改） ────────────┐  ┌─── 新增 ──────────────────┐
│                             │  │                            │
│  微信公众号                  │  │  通用数据源                 │
│  rss_poller (微信API)       │  │  news_collector            │
│      ↓                      │  │  (JSON API / RSSHub)       │
│  rss_store                  │  │      ↓                     │
│  (subscriptions/articles)   │  │  news_store (新表)          │
│      ↓                      │  │  (news_sources/news_items)  │
│  RSS XML 输出               │  │      ↓                     │
│                             │  │  AI 提取 → 日报生成         │
│                             │  │                            │
└─────────────────────────────┘  └────────────────────────────┘
```

### 9.3 对比

| | 微信 poller（现有） | 通用 news_collector（新增） |
|---|---|---|
| 数据源 | 微信 MP API | 各网站 JSON API + RSSHub |
| 认证 | 需要 token/cookie | 大部分不需要 |
| 标识 | fakeid | URL |
| 返回格式 | JSON（微信专有） | JSON（标准） |
| 存储表 | subscriptions / articles | news_sources / news_items |
